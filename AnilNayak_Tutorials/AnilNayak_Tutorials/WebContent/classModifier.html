Class Modifiers :

  • Whenever we are writing our own classes we have to provide some information about our classes to JVM .
  • This information including :
    • Whether this class can be accessible from anywhere or not.
    • Whether child class creation is possible or not .
    • Whether object creation possible or not for that class .
  • We can provide this information by declaring with appropriate modifiers .
  • The only applicable modifiers for outer class is :
    • public
    • default
    • final
    • abstract
    • strictfp
  • The only applicable modifiers for inner class is :
    • public
    • default
    • final
    • abstract
    • strictfp
    • private
    • protected
    • static
public classes:
  • If a class declared as public then we can access that class from anywhere either within or outside of package .
default classes:
  • If a class is default then we can access that class only only within the current package i.e, from outside package we can not access .
  • Hence default access is also known as package level access .
final :
  • final is the modifiers applicable for classes, methods and variables .
final method :
  • If we declared any class method as final, then we can not override that method in child class .
  • final method we can not override .
final class :
  • If a class declared as final, then we can not create child class .
  • The main advantage of final keyword is we can achieve security, means no one allowed to change the value or extend our final class .
  • But the main disadvantage of final keyword is we are missing key benefits of Object oriented features like [Inheritance by final class and polymorphsim by final methods]
  • If there is no specific requirements then it is never recommended to use final keyword .
abstract modifiers :
  • abstract modifiers applicable only for methods and classes, but not for variable .
abstract methods :
  • Eventhough we don't know about implementation, still we can declare a method with with abstract modifier i.e abstract method has only declaration but not implementation .
  • Hence abstract method declaration should compulsory ends up with semicolon .
Example: public abstract void t1(); [ correct abstract method ]
public abstract void t2(){}; [not correct abstract method ]
  • Child class is responsible to provide implementation for parent class abstract method .
  • The main advantage of placing abstract method in the parent class is we can provide guide lines to the child classes, such that which method compulsory child has to implement .
  • abstract method never talks about implementation .
  • The following are various illegal combinations of modifier for methods .
    • abstract with final
    • abstract with synchronized
    • abstract with static
    • abstract with native
    • abstract with private
    • abstract with strictfp
Example: public abstract final void m1(); Compiler Error : illegal combination of modifiers : abstract and final .
abstract class :
  • For any java class, if we don't want object creation such type of class we have to declare with abstract modifier i.e, for abstract class object creation is not possible .
abstract class vs abstract method :
  • If a class contains at least one abstract method then compulsory we should declare class as abstract, otherwise we will get compile time error .
  • Eventhough a class does not contain any abstract methods still we can declare a class as abstract . If we dont want to create object for that class, then we can declare that class as abstract .
Example: HttpServlet is a abstract class , but it does not contains any abstract methods.
abstract vs final:
  • For abstract method we should override in child class to provide implementation, but for final methods we can not override . Hence abstract final combination is illegal for method.
  • For abstract class we should create child class to provide implementation, but for final class we can not create child class . Hence abstract final combination is illegal for classes
  • We can not declare abstract method inside final class, but we can declare final method inside abstract class .
  • If we are extending any abstract class then for each and every abstract method of parent class we should provide implementation, other wise we have to will get compile time error .
