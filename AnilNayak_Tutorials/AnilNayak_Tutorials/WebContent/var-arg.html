Var-arg Method

  • Until jdk 1.4 version we can not declare a method with number of arguments, so when there is any change in number of arguments, then we should create a new method, so this approach increase the length of the code and reduce the readability of the code .
  • So to resolve this problem Sun people introduced var-arg method in jdk 1.5 version .
  • So from jdk 1.5 onwards we can declare a method with number of arguments, and this method is called var-arg method .
  • We can declare var-arg method as like this public m1(int...x)
  • We can invoke this method by passing any number of integer arguments .
    Example:: m1();
    m1(10);
    m1(10,20);
    m1(10,20,30)
Lets take an Example::
  • The correct way of writing var-arg method as follows below :
    m1(int...x)Right
    m1(int x...)Wrong
    m1(int ...x)Right
    m1(int. ..x)Wrong
    m1(int .x..)Wrong
  • We can write var-arg parameter with normal parameter, but var-arg parameter should be last parameter .
  • In var-arg method we can take only one var-arg parameter, we can't take more than one var-arg parameter .
  • var-arg method always get least priority, means when no method matched then var-arg method will got chance to execute .
  • We can replace single dimensional array with var-arg parameter, but reverse not possible .
    example:: m1(int[] x) == m1(int...x)Right
    m1(int...x) == m1(int[] x)Wrong
