  • Set

HashSet

  • The underlying data structure is HashTable .
  • Insertion order is not preserved and it is based on hashcode of objects.
  • Duplicate objects are not allowed and if we are trying to insert duplicate objects then we won't get any compile time error and Runtime error simply add() method returns false .
  • Null insertion is possible .
  • Heterogeneous objects are allowed .
  • HashSet implements Serializable amd Cloneable interfaces, but not RansomAccess interface .
  • If our frequent operation is search operation then HashSet is the best choice .
Constructors
  • HashSet h = new HashSet();
    creates an empty HashSet objects with default initial capacity 16 and default fill ratio or load factor 0.75
  • HashSet h = new HashSet(int initialCapacity);
    creates an empty HashSet objects with specified initial capacity and default fill ratio 0.75
  • HashSet h = new HashSet(int initialCapacity, float fillRatio);
  • HashSet h = new HashSet(Collection c)
Load Factor :
  • After loading this much factor a new HashSet object will be created, this factor is called Load Factor or Fill ratio .
