throw keyword

  • Sometimes we can create Exception object explicitly and we can hand over that object to the JVM manually, for this we have to use throw keyword .
Example: throw new ArithmeticException("/ by zero");
  • The result of following 2 programs exactly same .
  • Most of the times, we can use throw keyword for customized exceptions(Our own exceptions) but not for predefined Exceptions .
  • After throw statement we are not allowed to write any statement directly, other wise we will get Compile time Error saying, Unreachable Statement .
  • We can use throw keyword only for Throwable types, other wise we will get Compile Time Error: saying: incompatible types .
