Map Interface is implemented by HashMap, LinkedHashMap, WeakHashMap, IdentityHashMap, TreeMap, HashTable classes.
  • Map
    • HashMap
    • LinkedHashMap
    • WeakHashMap
    • IdentityHashMap
    • TreeMap
    • HashTable
  • Map is not child interface of Collection
  • If we want to represent a group of objects as a key and value pairs then we should go for Map
  • Both keys and values are objects only .
  • Duplicate keys are not allowed . But values can be duplicated .
  • Each key and value pair is called an Entry .
Map interface methods :
  • Object put(Object key, Object value)
    • To add one key-value pair, if the key is already available then old value will be replaced with new value.
  • void putAll(Map m)
  • Object get(Object key)
  • Object remove(Object key)
  • boolean containsKey(Object key)
  • boolean containsVlaue(Object value)
  • boolean isEmpty()
  • int size()
  • void clear()
  • Set keySet()
  • Collection values()
  • set entrySet()
Entry Interface :
  • Each key-value pair is called an Entry .
  • Without existing Map Object there is no chance of existing Entry object . Hence Entry interface is defined inside Map interface .

HashMap

  • The underlying data structure is HashTable .
  • Duplicate keys are not allowed . But values can be duplicated .
  • Heterogeneous Objects are allowed for both keys and values .
  • Insertion order is not preserved and it is based on HashCode of the keys .
  • Null is allowed for key(Only once) and allowed for values (any no of times )
  • How to get synchronized version of HashMap ?
    Ans: By default HashMap is non-synchronized . But we can get synchronized version of HashMap by using synchronizedMap() method of Collections class .

Constructors:-

  • HashMap m = new HashMap();
    Creates an empty HashMap object with default initial capacity 16 and default fill ratio 0.75 or 75% .
  • HashMap m = new HashMap(int initialCapacity)
  • HashMap m = new HashMap(int initialCapacity, float fillRatio)
  • HashMap m = new HashMap(Map m)
