Variables

A Java variable is the piece of memory name that contains a data value. Variables are basically used to store data for you java program and use that data in your program when you need .

Java Variables Types

Based on type variables are divided into two parts
  • Primitive Variables
    • e.g: int x = 10; (Here x is the primitive variable)
  • Reference Variables
    • e.g: Employee e = new Employee(); (Here e is the reference variable)
Based on the purpose and positions variables are divided into three parts
  • Instance variable
  • Static variable
  • Local variable
Instance Variable
  • If the value of the variable is varied from object to object is called instance variable .
  • For every object a separate copy of instance variable will create .
  • The scope of the instance variable is same as object, because the instance variable will create while creation of object and destroyed at the time of object destroyed .
  • Instance variable store in heap area .
  • Instance variable should declare inside the class, but out side the method, block or constructor .
  • Instance variable can not access from static area directly, we should access through object reference, but from instance area can access directly .
  • For instance variable no need to perform initialization explicitly, JVM will provide default values .
  • Instance variable is also called as object level variables or attributes .

Static Variable
  • If the value of the variable is not changed from object to object, then such type of variable should declared with static modifier and that variable is called as static variable .
  • For all object a single copy of static variable will create .
  • The scope of the static variable is same as class, static variable will be create at the time of class loading and destroyed at the time of class unloading .
  • Static variable stored in Method area .
  • Static variable should declare inside the class, but out side the method, block or constructor .
  • Static variable can access from both static and instance area directly, through class name or through object reference, but within the same class it is recommended to access directly.
  • For static variable no need to perform initialization explicitly, JVM will provide default values .
  • Static variable is also called as class level variables or attributes .

Local Variable
  • When some variable required to declared only for the particular block, constructor or method, then that type of variable is called as local variable .
  • Local variable stored in Stack area .
  • The scope of the local variable is depends upon the block, local variable executed when block execute and destroyed when block execution complete .
  • For local variable JVM won't provide any default value, we should perform initialization before using that variable, when we are not using that variable then initialization is not compulsory .
  • The only applicable modifier for local variable is final and default if we declared with any other modifier we will get compile time error .
