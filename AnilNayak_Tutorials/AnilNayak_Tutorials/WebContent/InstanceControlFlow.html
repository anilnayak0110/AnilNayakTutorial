Instance Control Flow :

  • Whenever we are creating an object the following sequence of events will be executed .
  • Execution of instance variable assignments and instance blocks from top to bottom .
  • Execution of constructor .
  • Note: static control flow is one time activity, which will be executed at the time of class loading. But instance control flow is not one time activity, for every object creation it will be performed .
Instance control flow in parent to child relationship :
  • Whenever we are creating child class objects the following sequence of events will be executed automatically .
  • 1. Identification of instance members from parent to child .
  • Execution of instance variable assignments and instance blocks only in parent class .
  • Execution of parent class constructor .
  • Execution of instance variable assignments and instance blocks in child class .
  • Execution of child class constructor .
  • Object creation is most costly operation in java. Hence if there is no specific requirement then it is never recommended to create object .
  • Usage of static methods is recommended when compared with instance methods .
