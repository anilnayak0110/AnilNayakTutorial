Java Coding Standards

Whenever we are writing java code, the real purpose of code should understand, so we must follow some coding standard while writing java code . By using java coding standard, the readability and maintainability of the code will increase . Lets see a sample Example:
Wrong Java coding standard

Correct Java Coding standard

The difference is every class first statement should start with some standard package name .
Coding Standard for classes and interfaces
Every class should start with uppercase character, and if it contains multiple words, then the inner word should start with uppercase character .
class name Example:
String
StringBuffer
SringBuilder
Employee
Student
StockMarket
interface name Example:
List
RandomAccess
Runnable
EmployeeService
StudentService
StockMarketService
Coding Standard for methods
Every method name should start with lowercase character, and if it contains multiple words, then the inner word should start with uppercase character (camel-case convention).
method name example:
run()
sleep()
println()
getEmployeeName()
getAddress()
Coding Standard for variables
Every variable name should start with lowercase character, and if it contains multiple words, then the inner word should start with uppercase character (camel-case convention).
variable name example:
name
address
mobileNumber
cityName
Coding Standard for constants
Every constants should start with only uppsercase character, and if it contains multiple words, then each word should separate with underscore symbol(_) .
constants name example:
MAX_VALUE
MIN_VALUR
URL_ADDRESS
